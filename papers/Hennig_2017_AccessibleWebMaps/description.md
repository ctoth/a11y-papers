Hennig, Zobl, and Wasserburger (2017) present differentiated design recommendations for making web maps accessible to three distinct visual impairment user groups -- color vision impaired, moderately visually impaired, and severely visually impaired/blind -- grounded in empirical questionnaire data from 158 and 129 respondents across two Austrian projects (AccessibleMap and senTour), covering user interface structure, map visual variables, interaction modes, and mandatory verbal content description. Key contributions include a requirements matrix by impairment type (Table 7), detailed visual variable recommendations for map features (Table 8), a best-practice analysis of existing accessible web maps (Table 6), and a four-concept accessibility framework (accessibility, usability, utility, compatibility) that argues technical compliance alone is insufficient. For this collection, the paper provides the most detailed implementation-level guidance on accessible web map design, directly complementing Biggs_2025_MapEquivalentPurpose's evaluation framework for text map representations and connecting to the broader web standards, visual impairment, and evaluation methods research areas.
