Martins and Duarte (2023) conduct an automated WCAG 2.1 accessibility evaluation of 2,884,498 web pages from 166,311 websites (March–September 2021), finding an average of 30 accessibility errors per page with only 0.5% of pages having zero errors, and identifying that the top failures are text contrast (79% of pages), missing accessible names (68%), and improper ARIA roles and properties (24%, via WCAG 1.3.1). Using Wappalyzer and SimilarTech to fingerprint 3,482 technologies across 32 categories, they apply Mann-Whitney U and Kruskal-Wallis tests to show that 29 of 32 technology categories have statistically significant relationships with the A3 accessibility metric, with specific technologies like Isotope.js (A3=0.328), jQuery (0.472), and React (0.613) associated with poor accessibility, while Python (0.816), Mustache JS (0.809), and Jimdo (0.855) correlate with better outcomes. For this collection, the paper provides the largest empirical baseline linking web technology choice to accessibility level, directly implicating improper ARIA usage as a measurable cause of 24% of all web pages failing WCAG 1.3.1, with a publicly available dataset (Zenodo DOI: 10.5281/zenodo.7494722) enabling further ARIA-specific analysis.
