Pool (2023) empirically compares nine integration-amenable automated web accessibility testing tools (alfa, axe-core, Continuum, Equal Access, HTML CodeSniffer, Nu Html Checker, QualWeb, Tenon, and WAVE) by running all nine against a 121-page corpus, finding that every pairwise tool combination is bidirectionally complementary — adding any second tool increases issue instance discovery by 13–767% and adds 15–51 distinct issue types, with each tool holding at least 7 issues it alone detects. The study maps the tools' 1,327 tests to 245 distinct accessibility issues aligned to WCAG 2.1, documents per-tool specializations (e.g., axe-core for color contrast and landmarks, WAVE for label clarity, Nu Html Checker for attribute validity), and enumerates 14 non-trivial engineering integration costs for multi-tool ensemble deployment. For this collection, the paper is the foundational empirical reference on automated evaluation methods: it quantifies why no single tool suffices, establishes which tools specialize in which issue categories, and frames the ensemble testing trade-off between discovery completeness and integration effort.
