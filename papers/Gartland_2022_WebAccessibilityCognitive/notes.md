# The State of Web Accessibility for People with Cognitive Disabilities: A Rapid Evidence Assessment

**Authors:** Sara Gartland, Paul Flynn, Maria Ana Carneiro, Greg Holloway, Jose de Sousa Fialho, Joe Cullen, Emma Hamilton, Amy Harris, Clare Cullen
**Year:** 2022
**Venue:** Behavioral Sciences (MDPI), Vol. 12, No. 2, Article 26
**DOI/URL:** https://doi.org/10.3390/bs12020026
**Funding:** European Commission, grant 101018048 (LIVE-IT Project)

## One-Sentence Summary

A rapid evidence assessment synthesizing 45 peer-reviewed studies (2006-2021) on web accessibility for people with cognitive disabilities, yielding a three-theme framework: barriers, tools, and methodological best practices.

## Problem Addressed

Despite decades of WCAG development, people with cognitive disabilities (PwCDs) continue to face severe web barriers. COVID-19 accelerated the digitization of daily life, making accessible web access urgently necessary for this marginalized group. A baseline systematic review was needed to inform the LIVE-IT Project's co-design labs (Ireland, Portugal, UK, Greece).

## Key Contributions

- Systematic synthesis of 45 empirical papers on cognitive accessibility (2006-2021), organized by three themes: barriers, tools, and research methodology
- Identification of the dominant barrier: text-heavy web content with insufficient non-text alternatives
- A four-quadrant best-practices framework for inclusive research: **People, Technology, Space, Process**
- Mapping of WCAG 2.0/2.1/2.2 criteria specifically identified by the W3C COGA Task Force as supporting PwCDs
- Documentation of which cognitive disabilities are most/least studied in current literature

## Methodology

- Rapid Evidence Assessment following PRISMA-P and registered with PROSPERO
- Databases searched: SCOPUS, ProQuest, Web of Science (50 search strings across 15-year window, 2006-2021)
- Initial pool: ~265,000+ records; after deduplication and screening: 45 included papers
- Quality appraisal: CASP (Critical Appraisal Skills Programme) checklists applied to all 45 papers; no papers excluded on quality grounds
- Data extraction and thematic analysis using PICO model
- Study types: primarily qualitative/quasi-experimental (N=31), some experimental (N=14)

## Key Findings by Theme

### Theme 1: Barriers to Web Accessibility

**Cross-disability barriers (affecting PwCDs broadly):**
- Excessive text-based content is the dominant barrier across all cognitive disability types
- Even WCAG-AA-compliant sites were rated poorly if content was not relevant or immediately useful to PwCDs [refs 30-33]
- Beyond usability: PwCDs need web tools that support community-building, identity expression, and combating social exclusion [34, 35]
- Social media tools that replicate offline social hierarchies are rated poorly [36-38]

**Broadly-defined cognitive disability populations (intellectual disabilities, learning disabilities, mild cognitive impairment):**
- Simplified text (images + familiar words + larger fonts) improved comprehension but reduced depth of information, creating a tradeoff [39]
- Image navigation increased task completion speed vs text-based navigation [41]
- Easy-to-read website design improved reading efficiency [40]
- Frustration from page load errors perceived as personal failures [42]
- Students with learning disabilities used Google as spell-checker before searching [43]

**Specific populations:**
- **Memory issues (dementia, aging, concussion):** Benefit from audio, video, picture-based information modes [44-46]
- **Autism:** Limited scrolling and direct navigation pathways improve experience [47, 48]
- **Dyslexia:** Most studied specific group (9 papers). Key barriers: font, amount of text, text colour, CAPTCHA password management, query formulation in academic databases [49-57]; non-text alternatives urgently needed
- **Down syndrome/cerebral palsy/Huntington disease:** Only 4 papers; usability of specific tools (Wii remote, Android apps); multimodal (audio + visual + haptic) information supports task completion [58-61]

### Theme 2: Tools

- 31 of 45 studies directly investigated websites/web apps/web-based tools for PwCDs
- 14 studies created novel tools; 17 evaluated existing tools
- **Consistent finding across tool types:** Too much text; need for more non-text options (real photos preferred over animated graphics; audio options)
- **Critical finding:** Adding non-text content that requires more scrolling is counterproductive - negates accessibility benefit [64]
- **Cognitive load from information search** is a systemic barrier: PwCDs pre-search terms for spelling, toggle between multiple sites, rely on image searches to avoid spelling errors [general finding across studies]
- Poorly rated existing tools: FindMyApps [32], Social Support Aid [45], ReACT [33]
- Co-design-developed tools produced more relevant, real-world-grounded outcomes

### Theme 3: Methodological Best Practices (Figure 5)

**People:**
- Participants must be co-designers, not test subjects
- Include caregivers as co-designers where appropriate
- Include professionals with expertise in specific cognitive disabilities
- Build relationships; plan for relationship-building time in study design

**Technology:**
- Select tools/platforms/devices relevant to target group (involve PwCDs in this choice)
- Provide training for all selected tools
- Pre-assess preferred device(s); consider multi-device access

**Space:**
- Ensure space is friendly and welcoming
- Position participants as co-designers in the space
- Use familiar spaces when possible (reduces cognitive load from environment)
- If unfamiliar space: provide familiarisation time

**Process:**
- Obtain informed consent (may require caregiver/parent/guardian consent)
- Use phased approach: Introduction & Training → Testing & Refining → Evaluation (built on participatory design principles)
- Plan multiple sessions; relationship-building across sessions yields better data
- Provide reflection opportunities after each session (oral, written, or video)

### Cognitive Disability Types and Corresponding Web Needs

| Disability Type | Primary Web Barrier | Key Intervention |
|----------------|---------------------|------------------|
| Intellectual disability | Text overload | Images, familiar words, larger fonts |
| Dyslexia | Font, text formatting, spelling | Text customization, spell support, audio |
| Autism | Excessive scrolling, complexity | Direct navigation, limited scrolling |
| Dementia/memory | Text-only information | Audio, video, picture-based content |
| Concussion/aging | Text-only information | Multimodal content |
| Down syndrome | Task navigation complexity | Multimodal (audio+visual+haptic) |
| Broadly defined CID | Cognitive search load | Pre-search support, image search options |

## Limitations

- Review does not directly address effectiveness of interventions (too few RCTs; mostly qualitative/quasi-experimental studies)
- Study population disaggregation was strict: many otherwise relevant papers excluded because cognitive disability results couldn't be separated from other disability data
- 60% of included studies focused on text-based websites; limits generalizability to modern web
- Geographic scope skewed toward EU/UK/North American research
- Health, commerce, and employment sectors under-represented in corpus
- Most studies used researcher-developed test websites, not real-world web navigation, which misses cross-site and multi-page navigation barriers

## Testable Properties

- WCAG AA compliance does not guarantee subjective accessibility ratings from PwCDs — user relevance and usefulness are independent dimensions
- Simplified text reduces comprehension barriers but reduces information depth — tradeoff is measurable
- Co-design studies (participants as designers) produce more ecologically valid findings than expert/automated testing
- Non-text alternatives that increase scrolling requirements negate accessibility benefits — scrolling load must be measured, not just content format
- Relationship between researcher and participant influences data quality for PwCDs — single-session studies produce lower-quality findings than multi-session designs

## Relevance to Project

This paper is the foundational evidence map for cognitive accessibility in the web context. It defines what barriers exist, which populations are affected, which WCAG criteria specifically matter for PwCDs, and what methodological standards should govern future research with this population. For the collection, it serves as the anchor reference for cognitive accessibility topic area — every future paper in this domain should be cross-referenced to this review.

## Open Questions

- [ ] What is the actual effectiveness (not just existence) of cognitive accessibility tools? Few papers provide rigorous outcome data
- [ ] How does cognitive accessibility interact with intersectional disabilities (e.g., autism + intellectual disability)?
- [ ] Health, commerce, and employment sectors are essentially unstudied — significant gap
- [ ] Non-text web content (audio, video, pictogram-heavy interfaces) for PwCDs is severely under-researched
