## Key Citations for Follow-up

**[8] Takagi, Asakawa, Fukuda, Maeda (ASSETS 2004)** — The full technical paper for aDesigner, "Accessibility Designer: Visualizing Usability for the Blind." This is the primary technical reference behind this paper's Section 5.2. Contains the reaching-time calculation methodology and full empirical evaluation of the tool. High priority for collection.

**[1] Asakawa & Itoh (ASSETS 1998)** — The original Home Page Reader user interface paper. Describes HPR architecture: HTML-tag parsing approach, number-pad navigation design, synchronization with Netscape Navigator. Foundational for screen reader / voice browser architecture. High priority for collection.

**[21] Asakawa & Takagi (ASSETS 2000)** — "Annotation-based Transcoding for Nonvisual Web Access." Web page restructuring pipeline for blind users — an early form of server-side accessibility adaptation. Relevant to transcoding and automated remediation research strand.

**[22] Takagi, Asakawa, Fukuda, Maeda (ASSETS 2002)** — "Site-wide Annotation: Reconstructing Existing Pages to be Accessible." Extends transcoding to entire sites; systematic approach to retroactive accessibility. Relevant to automated remediation.

**[14] Asakawa & Itoh (CHI 1999)** — "User Interface of Non-visual Table Navigation Method." Table navigation is identified in this paper as a major usability advance for blind web access (HPR 2.5 feature). Understanding how table navigation works in early screen readers is relevant to screen reader architecture.
